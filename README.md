# Inventory Chatbot

## State

Starting from the **"State"**, it contains the following:

- **message**: message history between the bot and the user  
- **question**: the question asked by the user or the input, can be either chitchat or a question about SQL query  
- **sql_query**: the SQL query generated by the LLM  
- **sql_result**: the result retrieved by the SQL query  
- **error**: if the LLM fails to generate a SQL query  
- **intent**: the intent of the user's message  

---

## Nodes

Then for the nodes, there are **six**:

### 1. intent_node
Given the **INTENT_PROMPT** and the human message, it will determine the intent as either **"CHITCHAT"** or **"generator"**.

### 2. chit_chat_node
Given the **CHITCHAT_PROMPT** and the human message, it will generate a chit-chat response without referring to the database.

### 3. sql_generator_node
Given the **SYSTEM_PROMPT** and the human message, it will generate a SQL query without referring to the database.

### 4. sql_executor_node
It fetches the **sql_query** from the state and executes it.

### 5. sql_corrector_node
It corrects and refines the SQL query if an error occurs in the state, until the query works.

### 6. responder_node
Given the **RESPONSE_PROMPT** and the SQL result, it will generate a response to the user.

---

## Flow Control Functions

After that, the graph was constructed. Two additional functions were created to help control the flow of the program:

- **executor_should_continue**: the function used in the conditional edge of the **sql_executor_node**. If it returns **"error"**, it should keep on calling the responder until it reaches a correct response and returns **"responder"**.

- **intent_should_continue**: the function used in the routing of the **intent_node**. If the intent is **"generator,"** it goes to the **sql_generator_node**. If it's **"chitchat,"** it goes to the **chit_chat_node**. It repeats this with every message.

# NEO4J Knowledge Graph Agent

## State

Contains the following:

- **message**: message history between the bot and the user  
- **question**: the question asked by the user or the input, can be either chitchat or a question about SQL query  
- **intent**: the intent of the user's message 
- **cypher**: generated cypher code
- **cypher_result**: the result retrieved by the cypher query
- **error**: if the LLM fails to generate a SQL query  

---

## Nodes

For the nodes, there are **seven**:

### 1. intent_node
Given the **INTENT_PROMPT** and the human message, it will determine the intent as **"add"**, **"update"**, **"inquire"**, **"delete"**.

### 2. execute_cypher
Executes Cypher query using LlamaIndex Neo4j Graph Store

### 3. synthesize_node
Synthesizes the cypher query based on the user's question.

### 4. add_node
Adds a new node to the graph

### 5. update_node
Updates an existing node in the graph

### 6. inquire_node
Inquires about a node in the graph

### 7. delete_node
Deletes a node from the graph
